# Code of Conduct

## Our Pledge

We want MiMo Bridge MCP to be a practical, welcoming project for people testing agentic coding workflows, Windows packaging, local MCP bridges, and low-token review loops.

Participants are expected to be respectful, constructive, and patient with people who have different experience levels.

## Expected Behavior

- Use welcoming and inclusive language.
- Assume good intent while still being precise about risks and bugs.
- Keep feedback focused on the work, not the person.
- Respect boundaries around privacy, local logs, credentials, and personal machine details.
- Help keep issues actionable by providing clear reproduction steps when possible.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Publishing private information without permission.
- Posting secrets, credentials, tokens, or private logs.
- Repeatedly derailing technical discussions.
- Pressuring maintainers or contributors to run unsafe commands or expose local services publicly.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; or block participants when needed to keep the project safe and productive.

If you see a problem, contact a maintainer through the repository's available GitHub channels.
